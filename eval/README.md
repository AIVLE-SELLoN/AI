# eval/ — 정량 실험

> ⚠️ **사람이 수동으로 실행한다. CI 에 넣지 말 것.**
> LLM 비용이 발생하고 수 분~수십 분 걸린다. `tests/` 와 성격이 완전히 다르다.

| | `tests/` | `eval/` |
| --- | --- | --- |
| 실행 | CI 가 PR 마다 자동 | **사람이 필요할 때만** |
| 목적 | 코드가 안 깨졌나 | 성능이 얼마나 나오나 |
| 비용 | 0 | **LLM 과금** |
| 데이터 | `tests/fixtures/` (소규모 예시) | `data/` (파이프라인 규모) |

## 실행

저장소 루트에서 실행한다 (`app` 을 import 하므로).

```bash
python eval/run_detection_eval.py
```

`data/` 가 먼저 생성돼 있어야 한다 — 생성기·데이터 확정 후 채워진다.

## 실험 4종

| 스크립트 | 무엇을 재나 | 규모 | 비용(추정) |
| --- | --- | --- | --- |
| `run_detection_eval.py` | 탐지율·오탐률·편중/전역 판정 정확도 | 채점 케이스 | **$0** (oracle) |
| `run_pipeline_eval.py` | 분류 오류가 탐지를 얼마나 깎는지 (①−②) | 시드 3벌 | ~$90 |
| `run_classify_eval.py` | 프롬프트1 aspect 분류 정확도 | 1,000건 × 반복 | ~$20 |
| `run_review_eval.py` | 프롬프트2 리뷰 aspect별 긍부정 | AI Hub 71603 | ~$20 |

**총예산 ~$220, 상한 $250로 관리** (mock data 정의서 기준 추정치).

## 비용 통제 4원칙

1. **반복 실험은 oracle 모드로** — 정답 라벨을 직접 입력해 LLM 을 안 태운다 ($0)
2. **분류는 배치 호출** — 건당 호출하면 비용 2배 + 속도 제한 초과
3. **분류 결과는 시드별 캐싱** — 같은 걸 두 번 돌리지 않는다
4. **모델 티어링** — 호출 많은 분류는 저가 모델, 호출 적고 중요한 개선안 생성만 상위 모델

## 원칙

- **`data/golden/` 은 `eval/` 만 읽는다.** `app/` 코드가 import 하면 컨닝이다.
- 실험 결과는 재현 가능해야 한다 — 시드·프롬프트 버전·모델명을 결과와 함께 기록할 것.

⚠️ 아래 스크립트들은 아직 **뼈대만** 있다. 데이터·스키마 확정 후 구현한다.
