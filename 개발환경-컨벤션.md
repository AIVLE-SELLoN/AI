## 3. 환경 세팅

```bash
python -m venv .venv
.venv\Scripts\activate          # Windows
source .venv/bin/activate       # macOS/Linux

pip install -r requirements.txt

cp .env.example .env            # LLM_API_KEY 채우기

uvicorn app.main:app --reload   # http://localhost:8000/docs
pytest                          # 10 passed 나와야 정상
```

**VSCode 쓰시면** 인터프리터를 `.venv` 로 선택하세요.
(`Ctrl+Shift+P` → `Python: Select Interpreter` → `.\.venv\Scripts\python.exe`)
안 하면 "Package not installed" 경고가 뜹니다.

## 4. 네이밍 컨벤션

기술 정리 문서의 네이밍 컨벤션과 동일. 자주 틀리는 것 위주로 정리.

### 파일·모듈

- 전부 `snake_case`, 단수형
- 모듈 내부 구성 통일: `router.py`(엔드포인트) / `service.py`(로직) / `prompts/`

### 함수

| 규칙 | 예 |
| --- | --- |
| `snake_case` + 동사로 시작 | `classify_aspect()`, `detect_anomaly()` |
| 내부 전용은 `_` 접두어 | `_build_prompt()`, `_parse_llm_response()` |
| bool 반환은 `is_`/`has_` | `_is_biased_channel()`, `_has_evidence()` |
| **LLM 호출은 `async def`** | `async def classify_aspect(...)` |

### 클래스

| 역할 | 접미사 | 예 |
| --- | --- | --- |
| API 입출력 | `~Request` / `~Response` | `ClassifyRequest` |
| 단계 간 데이터 | `~Result` / `~Item` | `AnomalyResult`, `ClassifiedItem` |
| 커스텀 예외 | `~Error` | `LlmParseError` |

### 상수·환경변수

- 상수: `UPPER_SNAKE_CASE`, **전부 `core/constants.py` 에**
- **매직넘버 금지** — `0.05` 대신 `constants.ALPHA`.
  정량 실험 때 바꿔가며 돌려야 하는 값이라 특히 중요.
- 환경변수: `UPPER_SNAKE` + `.env`, **API 키 하드코딩 절대 금지**

현재 정의된 상수:

| 상수 | 값 | 용도 |
| --- | --- | --- |
| `ALPHA` | 0.05 | 유의수준 |
| `MIN_SAMPLE_SIZE` | 10 | 이 미만이면 검정 skip |
| `MAX_RETRY` | 2 | LLM 재시도 (총 시도 = 1 + 2) |
| `COLLECTION_DETAIL_PAGES` | `"detail_pages"` | 컬렉션1 |
| `COLLECTION_REJECTION_REASONS` | `"rejection_reasons"` | 컬렉션2 |

### API 엔드포인트

- URL 은 `kebab-case` 소문자, **버전 접두어 필수**: `/api/v1/...`
- ML 연산 엔드포인트는 동사 허용 (`POST /classify` OK)

| 엔드포인트 | 담당 |
| --- | --- |
| `POST /api/v1/classify` | 현진 |
| `POST /api/v1/detect` | 서영 |
| `POST /api/v1/recommendations/generate` | 지인 |
| `POST /api/v1/reports`, `POST /api/v1/replies` | 용준 |
| `GET /health` | 공용 (k8s probe) |
| `GET /api/v1/*/ping` | 0주차 확인용 (구현 끝나면 삭제 가능) |

### 프롬프트 파일

- `{기능}_v{버전}.md` — `classify_aspect_v1.md`
- **구버전 삭제 금지.** 정량 실험이 버전 비교라서 그 자체가 실험 자산.
  개선할 때는 파일을 고치지 말고 `_v2.md` 를 새로 만드세요.
- 로딩은 `core/prompts.py` 의 `load_prompt(module, name)` 사용

### 기타

- 주석·docstring 한국어 OK, 함수명·변수명은 영어
- **로그에 추적 키(`cs_id`/`sku`) 항상 포함.**
  `llm_client` 의 `trace_key` 인자가 그 용도입니다.

## 5. 코드 작성 규칙

기술 정리 문서 6장 협업 규칙의 실무 버전.

1. **LLM 호출은 반드시 `core/llm_client.py` 경유.**
   각자 `openai` 를 직접 import 하지 마세요. 비용 집계와 재시도 정책이
   한 군데 모여야 합니다.
   ```python
   from app.core.llm_client import get_llm_client

   client = get_llm_client()
   data = await client.complete_json(prompt, trace_key=f"cs_id={cs_id}")
   ```

2. **모듈끼리 import 금지.** `app.detection` 이 `app.classification` 을
   import 하면 안 됩니다. 데이터는 `core/schemas.py` 의 Pydantic 모델로만.
   모든 모듈은 `core/` 만 import 합니다.

3. **`core/`·`schemas.py` 변경은 팀 채팅 선공지 → 이의 없으면 변경 + 영향받는 사람 태그.**

4. **fixture 로 개발.** `tests/fixtures/` 의 공용 Mock 데이터를 쓰면
   앞 단계가 안 끝나도 내 모듈을 시작할 수 있습니다.

5. **벡터DB 는 `get`/`query` 를 구분해서.**
   - `query()` — 의미 검색. "비슷한 내용" 찾을 때
   - `get()` — 메타데이터 완전일치. 키를 알 때. 임베딩 안 거쳐서 정확하고 쌈
